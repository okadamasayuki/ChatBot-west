# 会計相談チャット

LINE風のチャットUIで会計に関する質問ができるWebアプリです。Claude API が質問に回答し、AIで回答できない質問は会計の専門家にエスカレーションされます。

## 機能

- 💬 **LINE風チャットUI + トークルーム** — 相談を**質問ごとのルーム(トーク)**に分け、LINEのトーク一覧のように切り替えられる。「＋ 新規」で新しいルームを作成、一覧から過去の相談を再開できる
- 🗂 **「自分の相談」と「相談一覧」の切替** — 上部で切り替え。担当者は自分の相談のみ、財務(専門家)は全員分の「相談一覧」も閲覧可(相談者のメールも表示)
- 🛠 **財務向けの管理タブ** — 財務(専門家)でログインすると「内規」タブ(内規の編集・全員へ即時反映)と「マニュアル」タブ(社内マニュアルのテキスト/Markdownをアップロード)が表示される。内規・マニュアルは回答時にAIが参照する
- 🤖 **Claude API による自動回答** — 一般的な会計知識で答えられる質問はAIが即座に回答
- 📋 **内規の参照(非公開)** — 社内ルール(内規)を回答時にAIが参照。内規に該当する事項は一般的な会計知識より優先される(初期値: 「減価償却は定額法」)。**ワークスペースにつき内規は1つ**をチームで共通運用。**アプリからは編集不可(閲覧のみ)で、編集は管理者が Firebase コンソールで行う**。担当者(質問のみ)には内規は表示されない。Web上に平文公開はされない
- ❓ **不足情報の聞き返し** — 情報が足りず正確に回答できない場合は、AIが短い質問と選択肢ボタンを表示。質問者はボタンをタップするだけで答えられる(自由入力も可)
- 👤 **専門家エスカレーション** — AIが回答できない質問(個別の税務判断が必要な場合など)は、AIが回答方針の選択肢を生成して専門家タブに転送
- ✍️ **回答文(案)の自動生成** — 専門家が選択肢を選ぶと、Claude が回答文案を作成。専門家は編集のうえ「OK」ボタンで質問者に送信
- ⬇️ **Q&A履歴のダウンロード** — 質問と回答のペアを JSON / CSV(Excel対応・BOM付きUTF-8)でダウンロード可能
- ☁ **チーム共有(任意 / Firebase)** — Firebase を設定すると、内規・チャット・エスカレーション案件を**複数の担当者・端末でリアルタイム共有**できる。質問者と専門家を別々の端末で運用可能。未設定ならその端末のみで動作する
- 🔐 **担当者ログイン(Firebase Authentication)** — チーム共有時はメール＋パスワードのログインが必須。ログイン済みの担当者だけが Firestore を読み書きできる(合言葉との二段構え)
- 🧑‍💼 **役割の選択** — アカウント登録時に **担当者(質問のみ)** か **財務(会計の専門家)** を選択。担当者には「質問者」タブのみ、財務には「質問者」＋「専門家」タブが表示される。役割は Firestore の `members/{uid}` に保存される

## 使い方

本アプリは**常にクラウド(Firebase)に接続**して動作します。Firebase 設定はアプリに埋め込み済みのため、利用者は設定不要です。

1. GitHub Pages のURLを開く
2. **ログイン画面**で、管理者に発行されたアカウント(メール/パスワード)でログイン
3. 初回のみ Anthropic API キー(`sk-ant-...`)を設定(⚙)で入力
   - キーは [Anthropic Console](https://platform.claude.com/) で取得。ブラウザの localStorage にのみ保存されます
4. 「質問者」タブで会計の質問を入力(相談ごとにトークルームが分かれます)
5. AIが回答できない質問は「専門家」タブに届きます(財務ロールのみ表示)
6. 専門家は回答方針を選択 →「回答文(案)を作成」→ 内容を確認・編集 →「OK — 質問者に送信」
7. ヘッダーの「⬇ 履歴」からQ&A履歴をダウンロード

## チーム共有のセットアップ(Firebase)

複数の担当者で内規と質問⇄回答を共有する場合のみ必要です(無料枠で十分)。

1. [Firebase Console](https://console.firebase.google.com/) で新しいプロジェクトを作成
2. 左メニュー **Firestore Database** → **データベースの作成**(本番/テストどちらでも可)
3. 作成後、**ルール**タブを開き、以下に差し替えて「公開」(ログイン済みの担当者のみ読み書き可):
   ```
   rules_version = '2';
   service cloud.firestore {
     match /databases/{database}/documents {
       match /workspaces/{wid}/{document=**} {
         allow read, write: if request.auth != null;
       }
     }
   }
   ```
4. 左メニュー **Authentication** → **始める** → **Sign-in method** で **メール/パスワード** を有効化
5. **Authentication → Users → ユーザーを追加** で、担当者ごとのメールアドレスとパスワードを発行(アプリの「新規登録」ボタンからも作成可能)
6. プロジェクト設定(⚙)→ **全般** → 「マイアプリ」で **ウェブアプリ(</>)** を追加し、表示される `firebaseConfig` の `{ ... }` 部分をコピー
7. 本アプリを開き、設定(⚙)→「☁ チーム共有」に:
   - ①「Firebase 設定」へ 6 でコピーした `{ ... }` を貼り付け
   - ②「ワークスペース合言葉」へチーム共通の文字列(長めのランダム文字列推奨)を入力
   - 「保存」(自動で再読み込みされます)
8. ログイン画面が出るので、**役割(担当者=質問のみ / 財務=会計の専門家)を選んで**新規登録、または 5 で発行したアカウントでログイン。ヘッダーに **☁ (メールアドレス / 役割)** と表示されれば接続成功
   - 役割はアカウントごとに初回に決まり、以降は自動で反映されます(担当者は「質問者」タブのみ、財務は両タブが表示)
9. 同じ `firebaseConfig` と**同じ合言葉**を各担当者の端末にも設定し、それぞれのアカウントでログインすれば、内規・チャット・案件が共有される

### 内規の編集方法(Firebase コンソール)

内規はアプリからは編集できません。変更するには管理者が Firebase コンソールで直接編集します。

1. Firebase Console → **Firestore Database**
2. コレクション `workspaces` → 合言葉から生成されたドキュメントID(ワークスペースID)を開く
3. `naiki` フィールド(文字列)を編集して保存 → 全員にリアルタイム反映されます

> 初回はアプリ接続時に自動で `naiki: "## 会計処理ルール\n\n- 減価償却は定額法とする。"` が作成されます。ワークスペースIDが分かりづらい場合は、`workspaces` 直下のドキュメントは通常1つ(またはチームごとに1つ)なので該当のものを開いてください。

### 共有時のプライバシー / セキュリティ

- **二段構えのアクセス制御**: ①Firebase Authentication のログイン(`request.auth != null`)がないと Firestore を読み書きできない ②合言葉の SHA-256 をワークスペースID(Firestore のドキュメントID)にしているため、合言葉を知らない人は同じデータに到達できない
- 担当者ごとにアカウントを発行し、退職時などはコンソールから該当ユーザーを削除すればアクセスを無効化できます
- 部外者の勝手な新規登録を防ぎたい場合は、コンソールの Authentication 設定で自己登録を制限する(または本アプリの「新規登録」を使わず、管理者が Users からのみ発行する)運用にしてください
- ログアウトは設定(⚙)画面の「ログアウト」ボタンから。ログアウトすると共有データは画面から消えます
- `firebaseConfig` はクライアントに埋め込む前提の公開情報で、秘密ではありません(アクセス制御は Firestore ルールと Authentication で行います)

## アーキテクチャ

- 完全に静的な単一HTMLファイル(`index.html`)— ビルド不要、GitHub Pages でそのままホスティング
- ブラウザから Claude API(`claude-opus-4-8`)を直接呼び出し(CORS対応ヘッダー使用)
- 質問の振り分けには構造化出力(JSON Schema)を使用し、「回答」か「エスカレーション+選択肢生成」かをAIが判断
- APIキー・Firebase設定・合言葉などの設定はブラウザの localStorage に保存(端末ごと)
- チーム共有OFF時: チャット/案件/内規もその端末の localStorage に保存
- チーム共有ON時: 内規・チャット・案件は Firebase Firestore に保存され、`onSnapshot` でリアルタイム同期。APIキーは各端末の localStorage のまま(クラウドには送らない)

## 注意事項

- 本アプリは**チーム共有(Firebase)での利用を前提**としています。Firebase 未設定でも単体で動作しますが、これは初期設定用の暫定状態です
- ⚠️ APIキーはブラウザに保存されるため、**共有端末では使用しないでください**
- チーム共有時は、1つのワークスペースのデータ(相談ルーム・内規・案件)を**メンバー全員が共有**します(相談ルームも全員に表示されます)
- AIの回答は一般的な情報提供であり、税理士・公認会計士による助言に代わるものではありません
